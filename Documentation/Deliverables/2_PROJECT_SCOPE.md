# Project Scope Statement
The scope of the proposed project includes the development of a prototype Lounge Management 
System for FlyDreamAir’s Sydney and Melbourne lounges.

## In Scope: 
- Staff login 
- Passenger-facing booking access 
- Booking and cancellation/modification 
- Lounge entry verification 
- Membership tiers 
- Pay-per-use access 
- Occupancy tracking 
- QR code functionality 
- One year worth of historical booking records for audit logic 

## Out of Scope:
- Full loyalty integration
- Tracking of in-lounge purchases
- Complete privacy compliance implementation

# Project Constraints 

The project has some constraints that must be cautiously monitored in order to guarantee success.
1. Limited Prototype Timeframe: This project must be researched and prototyped within about 3 to 4 months, so the team may not have enough time to fully develop and refine every feature. Because of that, more advanced functions may be harder to complete within the semester timeline.

    > **Action:** Prioritize core features first, such as booking, lounge entry verification, membership handling, and basic occupancy management. Use a phased approach so advanced features can be left for later improvements.

2. Budget-Conscious Development: FlyDreamAir has a total budget of AU$1,000,000 over two years, the team is expected to save costs and not simply use the full amount. This means the proposed system must be practical and financially justified, not overly complex or expensive.

    > **Action:** Focus on business-critical functions only, choose cost-effective tools and technologies, and propose a phased delivery plan so the system can be built within a reasonable budget.
